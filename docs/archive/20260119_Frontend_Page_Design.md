## 1. 전체 페이지 구조 및 네비게이션

### 1.1 네비게이션 계층 구조

```
앱 시작
│
├─ [인증되지 않음]
│  ├─ 스플래시 화면 (Splash Screen)
│  ├─ 온보딩 슬라이드 (Onboarding Slides - 3페이지)
│  ├─ 로그인/회원가입 화면 (Auth Screen)
│  └─ 초기 정보 입력 화면 (User Info Input - 3단계)
│
└─ [인증됨] → 메인 앱
   │
   ├─ 하단 네비게이션 바 (Bottom Navigation - 3개 탭)
   │  ├─ Tab 1: 홈 피드 (Home Feed)
   │  ├─ Tab 2: 검색/탐색 (Explore) → 키워드 피드
   │  └─ Tab 3: 북마크 (My Places)
   │
   ├─ 모달/오버레이 화면
   │  ├─ AI 추출 바텀 시트 (URL Input)
   │  ├─ AI 처리 중 화면 (Processing)
   │  ├─ AI 결과 화면 (Extraction Result)
   │  └─ 장소 저장 바텀 시트 (Save Place)
   │
   └─ 상세 화면 (전체 화면 네비게이션)
      ├─ 장소 상세 화면 (Place Detail)
      ├─ 키워드 피드 화면 (Keyword Feed)
      ├─ 전체 화면 지도 (Full Map)
      └─ 이미지 갤러리 (Photo Gallery)
```

### 1.2 화면 전환 방식

| 시작 화면 | 목적지 화면 | 전환 방식 |
|----------|-----------|----------|
| 홈 피드 | 장소 상세 | Push (슬라이드) |
| 홈 피드 | 키워드 피드 | Push (슬라이드) |
| 홈 피드 | AI 추출 바텀 시트 | Modal (하단 슬라이드) |
| 키워드 피드 | 장소 상세 | Push (슬라이드) |
| 장소 상세 | 전체 지도 | Push (페이드) |
| 장소 상세 | 이미지 갤러리 | Push (줌 애니메이션) |
| 북마크 | 장소 상세 | Push (슬라이드) |
| 하단 탭 간 | 다른 탭 | 탭 전환 (크로스페이드) |

## 3. 페이지별 상세 설계

---

### 페이지 1: 스플래시 화면 (Splash Screen)

**목적**: 앱 초기 로딩 중 브랜드 인지도 제공

#### 화면 구성
```
┌─────────────────────────┐
│                         │
│                         │
│      [Mapsy Logo]       │ ← 중앙 정렬, 애니메이션
│      맵시 (Gradient)    │
│                         │
│                         │
│   "SNS 속 맛집을 찾다"   │ ← 서브 타이틀
│                         │
│                         │
└─────────────────────────┘
```

#### 구성 요소
1. **배경**: 그라데이션 (브랜드 컬러)
2. **로고**: 중앙 배치, 페이드인 + 스케일 애니메이션 (0.8초)
3. **서브 타이틀**: 로고 하단, 페이드인 (1초 지연)
4. **로딩 인디케이터**: 하단 중앙 (선택적)

#### 표시 시간
- 최소 2초 (애니메이션 완료)
- 최대 5초 (초기 데이터 로딩)

#### 다음 화면 분기
- **신규 사용자** → 온보딩 슬라이드
- **기존 사용자** → 홈 피드
- **로그아웃 상태** → 로그인/회원가입 화면

---

### 페이지 2: 온보딩 슬라이드 (Onboarding Slides)

**목적**: 신규 사용자에게 앱의 핵심 가치 전달

#### 화면 구성 (3페이지)

**페이지 1: AI 추출 기능**
```
┌─────────────────────────┐
│    [일러스트 이미지]      │ ← 상단 60%
│   SNS URL → AI 분석     │
│                         │
├─────────────────────────┤
│  SNS 속 맛집,           │ ← 제목 (H1)
│  AI가 찾아드려요         │
│                         │
│  인스타그램 릴스나       │ ← 부제목 (Body)
│  유튜브 영상 URL만       │
│  붙여넣으면 끝!          │
│                         │
│  ● ○ ○                 │ ← 페이지 인디케이터
│                         │
│  [건너뛰기]    [다음 →] │ ← 하단 버튼
└─────────────────────────┘
```

**페이지 2: 북마크 기능**
```
┌─────────────────────────┐
│    [일러스트 이미지]      │
│   폴더 정리 + 메모       │
│                         │
├─────────────────────────┤
│  저장하고, 정리하고,     │
│  공유하세요              │
│                         │
│  폴더별로 장소를 정리하고 │
│  메모와 별점을 남겨요     │
│                         │
│  ○ ● ○                 │
│                         │
│  [건너뛰기]    [다음 →] │
└─────────────────────────┘
```

**페이지 3: 피드 기능**
```
┌─────────────────────────┐
│    [일러스트 이미지]      │
│   트렌드 키워드 구름      │
│                         │
├─────────────────────────┤
│  트렌드 장소를           │
│  먼저 만나세요            │
│                         │
│  지금 뜨는 장소와         │
│  키워드를 한눈에          │
│                         │
│  ○ ○ ●                 │
│                         │
│  [건너뛰기]  [시작하기]  │
└─────────────────────────┘
```

#### 구성 요소
1. **일러스트**: 각 페이지별 개념 시각화 (Lottie 애니메이션 권장)
2. **제목**: H1 크기, 굵게, 2줄 이내
3. **부제목**: Body 크기, 3줄 이내
4. **페이지 인디케이터**: 점 3개, 현재 페이지 강조
5. **하단 버튼**: "건너뛰기" (텍스트), "다음" (Primary 버튼)
6. **스와이프**: 좌우 스와이프로 페이지 전환 가능

### 페이지 3: 로그인/회원가입 화면 (Auth Screen)

**목적**: Firebase OAuth를 통한 간편 로그인

#### 화면 구성
```
┌─────────────────────────┐
│                         │
│      [Mapsy Logo]       │ ← 상단 30%
│                         │
│                         │
├─────────────────────────┤
│                         │
│  [G] Google로 시작하기   │ ← Google OAuth
│                         │
│  [] Apple로 시작하기     │ ← Apple OAuth (iOS만)
│                         │
├─────────────────────────┤
│  ☑ 서비스 이용약관 및    │ ← 필수 체크박스
│     개인정보 처리방침     │
│     동의 (필수)          │
│                         │
│  ☐ 마케팅 정보 수신      │ ← 선택 체크박스
│     동의 (선택)          │
│                         │
│  [약관 보기]  [시작하기] │ ← 하단 버튼
└─────────────────────────┘
```

#### 구성 요소
1. **로고**: 상단 중앙 배치
2. **OAuth 버튼**:
   - Google: 흰색 배경, Google 로고 + 텍스트
   - Apple: 검정 배경, Apple 로고 + 텍스트 (iOS만 표시)
3. **약관 체크박스**:
   - 필수: 서비스 이용약관 + 개인정보 처리방침
   - 선택: 마케팅 정보 수신
4. **시작하기 버튼**: 필수 약관 동의 시에만 활성화

#### 유효성 검사
- 필수 약관 미동의 시 버튼 비활성화
- OAuth 실패 시 토스트 메시지: "로그인에 실패했어요. 다시 시도해주세요."

### 페이지 4: 초기 정보 입력 화면 (User Info Input)

**목적**: 신규 회원의 기본 정보 수집 (3단계)

#### 화면 구성 - Step 1: 생년월일

```
┌─────────────────────────┐
│  ← [이전]        1/3     │ ← 헤더 (단계 표시)
│  ━━━━━━━━━━━━━━━━━━━   │ ← 진행률 바 (33%)
│                         │
│  생년월일을 알려주세요    │ ← 제목 (H2)
│                         │
│  더 나은 추천을 위해      │ ← 설명 (Body)
│  필요해요                │
│                         │
│  ┌─────────────────┐    │ ← DatePicker
│  │  1995년 3월 15일 │    │
│  │   [날짜 선택]    │    │
│  └─────────────────┘    │
│                         │
│                         │
│        [다음 →]         │ ← 하단 버튼
└─────────────────────────┘
```

#### 화면 구성 - Step 2: 성별

```
┌─────────────────────────┐
│  ← [이전]        2/3     │
│  ━━━━━━━━━━━━━━━━━━━   │ ← 진행률 바 (66%)
│                         │
│  성별을 선택해주세요      │
│                         │
│  더 나은 추천을 위해      │
│  필요해요                │
│                         │
│  ┌─────────────────┐    │ ← 선택 카드 (3개)
│  │    👨 남성       │    │
│  └─────────────────┘    │
│  ┌─────────────────┐    │
│  │    👩 여성       │    │
│  └─────────────────┘    │
│  ┌─────────────────┐    │
│  │    🧑 기타       │    │
│  └─────────────────┘    │
│                         │
│        [다음 →]         │
└─────────────────────────┘
```

#### 화면 구성 - Step 3: 닉네임

```
┌─────────────────────────┐
│  ← [이전]        3/3     │
│  ━━━━━━━━━━━━━━━━━━━   │ ← 진행률 바 (100%)
│                         │
│  닉네임을 정해주세요      │
│                         │
│  다른 사용자에게          │
│  보여질 이름이에요        │
│                         │
│  ┌─────────────────┐    │ ← 텍스트 입력
│  │  맵시러버        │    │
│  └─────────────────┘    │
│  ✓ 사용 가능한 닉네임    │ ← 실시간 검증 메시지
│                         │
│  • 2-12자 이내           │ ← 규칙 안내
│  • 특수문자 제외         │
│                         │
│      [시작하기]          │
└─────────────────────────┘
```

#### 구성 요소
1. **헤더**:
   - 좌측: 이전 버튼 (Step 1에서는 숨김)
   - 우측: 단계 표시 (1/3, 2/3, 3/3)
2. **진행률 바**: 상단 고정, 33% → 66% → 100% 애니메이션
3. **제목**: H2 크기, 각 단계별 질문
4. **설명**: Body 크기, 회색, 2줄 이내
5. **입력 영역**:
   - Step 1: DatePicker (Bottom Sheet)
   - Step 2: 선택 카드 (3개, 단일 선택)
   - Step 3: TextField (실시간 중복 검사)
6. **하단 버튼**:
   - Step 1-2: "다음"
   - Step 3: "시작하기"

#### 유효성 검사
- **Step 1**: 날짜 선택 필수, 만 14세 이상만 가능
- **Step 2**: 성별 선택 필수
- **Step 3**:
  - 2-12자 이내
  - 중복 검사 (API 호출, 500ms 디바운스)
  - 특수문자 제외 (정규식 검증)

### 페이지 5: 홈 피드 화면 (Home Feed)

**목적**: 트렌드 장소와 키워드를 통한 발견 경험 제공

#### 화면 구성 (스크롤 가능)

```
┌─────────────────────────┐
│ [Logo]         [🔔 알림] │ ← 헤더 (고정)
├─────────────────────────┤
│                         │
│ ┌─ 섹션 1: 최신 장소 ───┐│ ← 무한 스크롤 영역
│ │  🆕 방금 추가된 장소  ││
│ │                      ││
│ │ ┌──────────────────┐ ││ ← 장소 카드 (세로 리스트)
│ │ │ [썸네일 16:9]     │ ││
│ │ │ 카페 온더무드      │ ││
│ │ │ 서울 종로구...    │ ││
│ │ │ #성수동 #카페     │ ││
│ │ │ 💾 45명 저장      │ ││
│ │ └──────────────────┘ ││
│ │                      ││
│ │ ┌──────────────────┐ ││
│ │ │ [썸네일]          │ ││
│ │ │ 장소명...         │ ││
│ │ └──────────────────┘ ││
│ └──────────────────────┘│
│                         │
│ ┌─ 섹션 2: 인기 피드 ───┐│ ← 가로 스크롤
│ │  🔥 지금 핫한 장소    ││
│ │                      ││
│ │ [썸네일] [썸네일] [썸네일]│ ← 그리드 3열
│ │  장소1    장소2    장소3 │
│ └──────────────────────┘│
│                         │
│ ┌─ 섹션 3: 내 TOP ──────┐│ ← 가로 스크롤
│ │  ⭐ 내가 저장한 인기   ││
│ │                      ││
│ │ [카드] [카드] [카드]  ││ ← 최대 10개
│ └──────────────────────┘│
│                         │
│ ┌─ 섹션 4: 트렌드 키워드┐│
│ │  📈 요즘 뜨는 키워드  ││
│ │                      ││
│ │ [#성수동] [#브런치]   ││ ← 칩 형태, 2줄
│ │ [#감성카페] [#데이트] ││
│ └──────────────────────┘│
│                         │
└─────────────────────────┘
         [+ FAB]            ← 플로팅 액션 버튼
```

#### 섹션별 구성 요소

**헤더 (고정)**
- 좌측: Mapsy 로고 (텍스트)
- 우측: 알림 아이콘 (빨간 점 배지 - 새 알림 시)

**섹션 1: 최신 추가 장소** (무한 스크롤)
- 제목: "🆕 방금 추가된 장소"
- 카드형 리스트 (세로 스크롤)
- 각 카드:
  - 썸네일 이미지 (16:9, 최대 높이 200px)
  - 장소명 (1줄, 굵게, 말줄임)
  - 주소 (1줄, 회색, 말줄임)
  - 키워드 태그 (칩 형태, 최대 3개)
  - 저장 횟수 (아이콘 + 숫자)
- 페이지네이션: 커서 기반, 페이지당 20개
- 로딩 상태: 스켈레톤 UI (카드 3개)

**섹션 2: 인기 인스타 피드** (가로 스크롤)
- 제목: "🔥 지금 핫한 장소"
- 그리드형 썸네일 (3열)
- 각 썸네일:
  - 정사각형 이미지 (1:1)
  - 하단 오버레이: 장소명 (1줄, 흰색 텍스트, 그림자)
  - 좌측 상단 배지: "🔥 인기" (가중치 상위 30%)
- 최대 30개 표시

**섹션 3: 내 저장 장소 TOP** (가로 스크롤)
- 제목: "⭐ 내가 저장한 인기 장소"
- 카드형 (가로 스크롤)
- 각 카드:
  - 썸네일 이미지 (4:3)
  - 장소명 (1줄)
  - 내 별점 (별 아이콘 1-5개)
- 최대 10개 표시
- 미로그인 시: "로그인하고 내 장소를 저장하세요" 버튼 표시

**섹션 4: 떠오르는 키워드 트렌드**
- 제목: "📈 요즘 뜨는 키워드"
- 태그 칩 형태 (가로 스크롤 또는 래핑, 2줄)
- 각 칩:
  - 키워드 텍스트 (예: "#성수동카페")
  - 트렌드 상승률 (예: "+120%", 빨간색 작은 텍스트)
  - 관련 장소 개수 (예: "25개", 회색)
- 최대 20개 표시
- 클릭 시 → 키워드 피드 화면

**플로팅 액션 버튼 (FAB)**
- 우측 하단 고정
- "+" 아이콘
- Primary 색상, 그림자
- 클릭 시 → AI 추출 바텀 시트

#### 상태 관리 필요 항목
- 각 섹션별 로딩 상태 (isLoading)
- 커서 페이지네이션 상태 (nextCursor, hasNext)
- 에러 상태 (error)
- 인증 상태 (isAuthenticated)

### 페이지 6: 탐색/검색 화면 (Explore)

**목적**: 키워드 검색 및 카테고리별 장소 탐색

#### 화면 구성

```
┌─────────────────────────┐
│ [🔍 검색창]       [필터] │ ← 헤더 (고정)
├─────────────────────────┤
│                         │
│ ┌─ 인기 카테고리 ────────┐│
│ │ [🍰 카페] [🍴 레스토랑]││ ← 카테고리 칩
│ │ [🏛 관광지] [🎨 문화] ││
│ └──────────────────────┘│
│                         │
│ ┌─ 인기 키워드 ──────────┐│
│ │  #성수동카페 (120)    ││ ← 키워드 리스트
│ │  #강남맛집 (95)       ││
│ │  #부산여행 (87)       ││
│ └──────────────────────┘│
│                         │
│ ┌─ 최근 검색 ────────────┐│
│ │  성수동 카페 [X]      ││ ← 최근 검색어
│ │  강남 맛집 [X]        ││
│ │  [전체 삭제]          ││
│ └──────────────────────┘│
│                         │
└─────────────────────────┘
```

#### 구성 요소

**헤더 (고정)**
- 검색창:
  - 좌측: 돋보기 아이콘
  - 입력: "장소, 키워드 검색..."
  - 우측: 지우기 버튼 (X)
- 필터 아이콘: 클릭 시 필터 바텀 시트

**인기 카테고리**
- 칩 형태, 가로 스크롤 또는 래핑
- 아이콘 + 텍스트
- 클릭 시 → 해당 카테고리 키워드 피드

**인기 키워드**
- 리스트 형태
- 각 항목:
  - 키워드 텍스트
  - 관련 장소 개수 (괄호 안)
  - 트렌드 상승 아이콘 (↗️)
- 클릭 시 → 키워드 피드 화면

**최근 검색**
- 리스트 형태
- 각 항목:
  - 검색어 텍스트
  - 삭제 버튼 (X)
- 전체 삭제 버튼

#### 검색 동작
- 검색어 입력 시 (300ms 디바운스)
- API 호출하여 자동완성 제안 표시
- 검색어 클릭 또는 엔터 → 키워드 피드 화면 (검색 결과)

### 페이지 7: 북마크 관리 화면 (My Places)

**목적**: 저장한 장소 관리 및 정리

#### 화면 구성

```
┌─────────────────────────┐
│ ☰ 내 저장 장소  [🔍][⚙] │ ← 헤더
├─────────────────────────┤
│ [전체] [가고싶은곳] [가본곳]│ ← 탭 바 (스크롤 가능)
│   15      8        3     │    (장소 개수)
├─────────────────────────┤
│ [최근순▼] [방문함 ☐]     │ ← 정렬/필터 바
├─────────────────────────┤
│                         │
│ ┌──────────────────┐    │ ← 장소 카드 (세로 리스트)
│ │ [썸] 카페 온더무드 [⋮]│
│ │      서울 종로구   ⭐⭐⭐│
│ │      "데이트 코스" ✓방문│
│ └──────────────────┘    │
│                         │
│ ┌──────────────────┐    │
│ │ [썸] 장소명       [⋮]│
│ │      주소          ⭐⭐│
│ │      "메모..."        │
│ └──────────────────┘    │
│                         │
│     ... (무한 스크롤)    │
│                         │
└─────────────────────────┘
```

#### 구성 요소

**헤더**
- 좌측: 메뉴 아이콘 (드로어 열기)
- 중앙: "내 저장 장소" 타이틀
- 우측: 검색 아이콘, 설정 아이콘

**탭 바**
- 가로 스크롤 가능
- 기본 탭:
  - 전체 (전체 저장 장소)
  - 가고 싶은 곳 (미방문)
  - 가본 곳 (방문함)
- 커스텀 폴더 (사용자 생성)
- 각 탭에 장소 개수 표시

**정렬/필터 바**
- 정렬 드롭다운:
  - 최근 저장순 (기본)
  - 이름순
  - 별점순
  - 방문 날짜순
- 필터 토글:
  - 방문함 / 미방문

**장소 카드 (리스트)**
- 좌측: 썸네일 (1:1, 60x60px)
- 중앙:
  - 장소명 (1줄, 굵게)
  - 주소 (1줄, 회색, 말줄임)
  - 별점 (별 아이콘 1-5개)
  - 메모 미리보기 (1줄, 회색, 인용부호)
- 우측:
  - 방문 여부 배지 ("✓ 방문함", 초록색)
  - 메뉴 아이콘 (⋮)
- 스와이프 액션:
  - 좌측 스와이프: 삭제 버튼 (빨간색)
  - 우측 스와이프: 메모 편집 버튼 (파란색)

**바텀 시트 메뉴** (메뉴 아이콘 클릭 시)
- "메모 수정"
- "별점 변경"
- "방문 날짜 기록"
- "폴더 이동"
- "삭제" (빨간색 텍스트)

**빈 상태**
- 일러스트 + "아직 저장한 장소가 없어요"
- "장소 추가하기" 버튼 → AI 추출 바텀 시트

### 페이지 8: 장소 상세 화면 (Place Detail)

```

#### 구성 요소

**헤더 이미지**
- 대표 이미지 (전체 너비, 16:9)
- 좌측 상단: 뒤로가기 버튼 (반투명 원형 배경)
- 우측 상단: 공유 버튼 (반투명 원형 배경)
- 하단 오버레이: 장소명 (흰색 텍스트, 그림자)

**기본 정보**
- 카테고리 (아이콘 + 텍스트)
- 평점 (별 + 숫자 + 리뷰 개수)
- 주소 (클릭 시 복사 토스트)
- 전화번호 (클릭 시 전화 앱 연동)

**영업 정보**
- 현재 상태: "영업 중" (초록색) / "영업 종료" (빨간색)
- 오늘 영업시간
- 확장 버튼: 클릭 시 요일별 영업시간 표시 (아코디언)

**위치 정보**
- 지도 미리보기 (축소, 클릭 시 전체 화면)
- 길찾기 버튼 (3개):
  - 카카오맵으로 길찾기
  - 네이버맵으로 길찾기
  - Google Maps로 길찾기

**관련 키워드**
- 키워드 칩 (가로 스크롤 또는 래핑)
- 클릭 시 → 키워드 피드 화면

**추가 이미지**
- 제목: "사진 (N개)"
- 썸네일 그리드 (3열, 최대 9개)
- "더보기" 버튼 (10개 이상 시) → 이미지 갤러리

**출처 콘텐츠**
- 제목: "이 장소를 언급한 게시물"
- 콘텐츠 카드 (가로 스크롤, 최대 10개):
  - 썸네일 이미지
  - 플랫폼 아이콘 (Instagram/YouTube)
  - 게시자 이름
  - 클릭 시 외부 브라우저로 원본 URL

**하단 고정 버튼**
- 좌측: "💾 저장하기" (Primary 버튼)
- 우측: "🔗 공유" (Secondary 버튼)
- 이미 저장된 경우: "✓ 저장됨" (회색, 비활성화)

- 관련 장소 개수 (예: "25개 장소")

- 최신순 (기본 선택, createdAt desc)
- 인기순 (saveCount desc)
- 거리순 (위치 권한 필요, 내 위치 기준)

**그리드뷰 모드** (2열)
- 정사각형 썸네일

**리스트뷰 모드**
- 좌측: 썸네일 (4:3, 80px 높이)
- 중앙:
  - 장소명 (1줄, 굵게)
  - 주소 (1줄, 말줄임)
  - 키워드 태그 (최대 3개)
  - 저장 횟수 + 평점
- 우측: 저장 아이콘

**페이지네이션**
- 무한 스크롤
- 페이지당 20개
- 스켈레톤 UI (로딩 시)

### 페이지 10: AI 추출 플로우 (3개 서브 화면)

**목적**: SNS URL에서 장소 추출 및 저장

#### 10-1: URL 입력 바텀 시트

```
┌─────────────────────────┐
│                         │
│  장소 추출하기           │ ← 제목
│                         │

│  URL을 붙여넣으세요      │
│                         │

- 제목: "장소 추출하기"

  - URL 유효성 검사 후 활성화
  - 클릭 시 → 처리 중 화면

  - "콘텐츠 다운로드 중..."
  - "영상 분석 중..."
  - "장소 추출 중..."
- 예상 시간 안내
- 취소 버튼

**폴링 방식**
- 2초마다 API 호출 (`GET /api/content/{contentId}`)
- status 확인:
  - PENDING/PROCESSING → 대기
  - COMPLETED → 추출 완료 화면
  - FAILED → 추출 실패 화면

## 7. 추가 고려사항

### 7.1 접근성 (Accessibility)

1. **시맨틱 레이블**: 모든 아이콘 버튼에 Semantics 위젯 적용
2. **최소 터치 영역**: 44x44px 이상
3. **색상 대비**: WCAG AA 기준 (4.5:1 이상)
4. **스크린 리더**: VoiceOver/TalkBack 지원

### 7.2 다국어 지원 (i18n)

**권장 패키지**: flutter_localizations, easy_localization

#### 지원 언어 (1차 MVP)
- 한국어 (ko_KR) - 기본
- 영어 (en_US) - 선택적

### 7.3 에러 처리

#### 네트워크 에러
- 연결 실패: "인터넷 연결을 확인해주세요" + 재시도 버튼
- 타임아웃: "요청 시간이 초과되었어요" + 재시도 버튼
- 서버 에러: "일시적인 오류가 발생했어요" + 재시도 버튼

#### 인증 에러
- 토큰 만료: 자동 갱신 (Refresh Token)
- 갱신 실패: 로그인 화면으로 이동

#### 유효성 검사 에러
- 실시간 피드백 (입력 중)
- 에러 메시지: 입력 필드 하단 빨간색 텍스트

### 7.4 로딩 상태 (Loading States)

#### 스켈레톤 UI
- 홈 피드: 카드 3개 스켈레톤
- 장소 리스트: 리스트 아이템 5개 스켈레톤
- 장소 상세: 전체 레이아웃 스켈레톤

#### 인피니트 스크롤 로딩
- 하단에 작은 원형 인디케이터

#### 전체 화면 로딩
- 중앙 원형 인디케이터 (스플래시, 인증)

### 7.5 빈 상태 (Empty States)

#### 홈 피드
- 일러스트 + "아직 장소가 없어요" + "첫 장소를 추가해보세요" 버튼

#### 북마크
- 일러스트 + "저장한 장소가 없어요" + "장소 추가하기" 버튼

#### 검색 결과
- 일러스트 + "검색 결과가 없어요" + "다른 키워드로 시도해보세요"

#### 임시 저장
- 일러스트 + "확인할 장소가 없어요" + "새로운 URL을 추가해보세요"
